\doxysubsection{Detailed Description}
Perform simple k\+NN classifier For each single new point, the computation complixity is O(\+Klog\+N). 

If there are M new points, O(\+MKlog\+N)

To perform a formal computational complexity analysis, let\textquotesingle{}s analyze the three methods in your code\+: {\ttfamily simple\+KNNClassifier}, {\ttfamily Experiment score}, and {\ttfamily Experiment confusion matrix}.

Complexity Analysis We will focus on the time complexity (T(n)) and space complexity (S(n)) for each method. Since we are assuming the worst-\/case scenario, we\textquotesingle{}ll analyze the method in terms of the input size \textquotesingle{}n,\textquotesingle{} which could represent the number of data points or features, depending on the context.

{\bfseries{1. {\ttfamily simple\+KNNClassifier} Method\+:}}


\begin{DoxyItemize}
\item {\bfseries{Time Complexity (T(n))\+:}}
\item In the {\ttfamily fit} method, it computes the variance for each column, which takes O(n) time for each column.
\item In the {\ttfamily dist\+\_\+sqr} method, it computes the squared distance for each data point against every training data point, resulting in O(n\texorpdfstring{$^\wedge$}{\string^}2) time complexity.
\item In the {\ttfamily \+\_\+pred} method, it sorts the distances, which takes O(\+K \texorpdfstring{$\ast$}{*} log(n)) time. It also counts the label frequencies, which takes O(\+K) time.
\item In the {\ttfamily test} method, if there are M new points, and for each point, it calls {\ttfamily \+\_\+pred}, the overall time complexity is O(M \texorpdfstring{$\ast$}{*} K \texorpdfstring{$\ast$}{*} (log(n) + K)).
\item {\bfseries{Space Complexity (S(n))\+:}}
\item The space complexity is mainly determined by storing the training data and labels. It uses additional space for variables like {\ttfamily res}, {\ttfamily heap}, and {\ttfamily cnt}.
\end{DoxyItemize}

{\bfseries{Justification\+:}}


\begin{DoxyEnumerate}
\item The time complexity of {\ttfamily simple\+KNNClassifier} depends on the number of new data points (M), the number of neighbors (K), and the number of training data points (n). In the worst case, it needs to calculate distances for each new point against all training points, making the time complexity O(\+M \texorpdfstring{$\ast$}{*} K \texorpdfstring{$\ast$}{*} n). 
\end{DoxyEnumerate}
